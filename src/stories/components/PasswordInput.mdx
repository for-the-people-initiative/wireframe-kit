# PasswordInput

A password input component with visibility toggle, strength indicator, criteria checklist, and optional confirmation field.

## Features

- **Show/hide toggle** — Eye icon to toggle password visibility
- **Strength indicator** — Visual bar that changes color based on password strength (weak → fair → strong → very strong)
- **Criteria checklist** — Optional list showing which password requirements are met
- **Confirm mode** — Second input field for password confirmation with match indicator

## Usage

```vue
<script setup>
import { ref } from 'vue';

const password = ref('');
</script>

<template>
  <PasswordInput
    v-model="password"
    label="Password"
    show-strength
    show-criteria
  />
</template>
```

## When to use

✅ **Use PasswordInput when:**
- Creating account registration forms
- Password change/reset flows
- Any form requiring password entry with feedback
- You need to show password strength to users

❌ **Don't use when:**
- Simple login forms where strength isn't needed (use InputText with type="password")
- Non-password sensitive data entry

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `modelValue` | `string` | `''` | v-model binding for the password value |
| `showStrength` | `boolean` | `true` | Show the strength indicator bar |
| `showCriteria` | `boolean` | `false` | Show the criteria checklist |
| `minLength` | `number` | `6` | Minimum password length for criteria |
| `requireStrength` | `'none' \| 'weak' \| 'fair' \| 'strong'` | `'none'` | Require password to meet minimum strength |
| `errorMessage` | `string` | `undefined` | Custom error message when requirement not met |
| `confirmValue` | `string` | `undefined` | Value for confirmation field (enables confirm mode when set) |
| `placeholder` | `string` | `'Enter password'` | Placeholder for main input |
| `confirmPlaceholder` | `string` | `'Confirm password'` | Placeholder for confirmation input |
| `label` | `string` | `undefined` | Label text above the input |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Input size |
| `isDisabled` | `boolean` | `false` | Disable the input |
| `isInvalid` | `boolean` | `false` | Show invalid state |

## Events

| Event | Payload | Description |
|-------|---------|-------------|
| `update:modelValue` | `string` | Emitted when password value changes |
| `update:confirmValue` | `string` | Emitted when confirmation value changes |
| `strength-change` | `'weak' \| 'fair' \| 'strong' \| 'very-strong'` | Emitted when password strength level changes |
| `valid` | `boolean` | Emitted when overall validity changes (strength + confirm match) |
| `focus` | `{ originalEvent: FocusEvent }` | Emitted on input focus |
| `blur` | `{ originalEvent: FocusEvent }` | Emitted on input blur |

## Slots

| Slot | Description |
|------|-------------|
| `error` | Custom error message content when strength requirement not met |

## Strength Levels

The password strength is calculated based on:
- Password length (≥ minLength, ≥ 10 characters)
- Contains uppercase letter
- Contains lowercase letter
- Contains number
- Contains special character

| Level | Score | Color |
|-------|-------|-------|
| Weak | 0-2 | Red |
| Fair | 3 | Orange |
| Strong | 4-5 | Green |
| Very Strong | 6 | Green |

## Accessibility

- The toggle button has proper `aria-label` that changes based on visibility state
- Input has `aria-describedby` linking to strength and criteria elements when shown
- Criteria items visually indicate met/unmet state with icons
